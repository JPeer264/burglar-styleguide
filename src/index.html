Methodology

Code Style

This styleguide follows the airbnb conventions for writing CSS. Here is a summary of the most important points:
  • Use soft tabs (2 spaces) for identation
  • Prefer dashes to camelCase when naming classes
  • Don't use ID selectors
  • When using multiple selectors, give each one an individual line
  • Only use line comments (in SASS)
  • Write detailed comments for code that is not self-explaining
              
  .navigation {
    width: 100%

    // this is a comment
    ul {
      float: right;
      list-style: none;
      margin: 0;
      padding: 0;
    }
  }

Class Naming Convention

For class naming this styleguide follows the BEM syntax. Here is a summary of the most important points:
  • Block: Standalone entity that is meaningful on its own. Eg. menu
  • Element: Parts of a block and have no standalone meaning. They are semantically tied to its block. Eg. menu__item
  • Modifier: Flags on blocks or elements. Use them to change appearance or behavior. Eg. menu__item--disabled
              
          .list {

            &__item {
              font-size: 1.2rem;

              &--fries::before {
                content: '🍟';
              }
            }
          }

Components

Button

A set of normal and full-width buttons are provided. For normal actions the regular button is to be used, for forms or actions that are completed, the done button should be used. To alert the user that the danger button should be used.

Normal

            
<button class="button"> Click me !</button>
<button class="button button--done"> Ok! </button>
<button class="button button--danger"> Cancel! </button>

Large

            
<button class="button button--fullwidth"> More Information </button>
<button class="button button--fullwidth button--done"> Register! </button>
<button class="button button--fullwidth button--danger"> Delete Order! </button>

Forms

Use fieldsets to seperate areas of concern. Use the * to show that a field is required and always provide a placeholder text to let the user know what should be typed in. Also provide labels for every input field. 
                
<form class="form">
    <fieldset class="form__fieldset">
        <div class="form__input-group">
            <label for="optional-field" class="form__label">Optional Field:</label>
            <input type="text" name="optional-field" id="optional-field" class="form__input-field form__input-field--optional" placeholder="John Doe">
        </div>

        <div class="form__input-group">
            <label for="required-field" class="form__label">Required Field:</label>
            <input type="text" name="required-field" id="required-field" class="form__input-field form__input-field--required" placeholder="Hauptstr. 5" required>
        </div>

        <div class="form__input-group">
            <label for="invalid-field" class="form__label">Invalid Field:</label>
            <input type="text" name="invalid-field" id="invalid-field" class="form__input-field form__input-field--invalid" placeholder="5020">
        </div>

        <div class="form__input-group">
            <label for="valid-field" class="form__label">Valid Field:</label>
            <input type="text" name="valid-field" id="valid-field" class="form__input-field form__input-field--valid" placeholder="Salzburg">
        </div>
    </fieldset>
</form>
